### Why ordering matters

`logAudit()` in `src/lib/auditLog.ts` has 26 callers in `src/`. Several deliberately write `action: 'create'` as a *domain verb* (e.g., `useLessonForm.ts:803` writes `'create'` for entity_type `'lesson'` — meaning "lesson was scheduled by the user"). This is distinct from CRUD-trigger `'insert'`.

If we backfill `entity_type` first (plural → singular) and then `action` second, the action backfill cannot distinguish trigger-emitted rows (which now have singular entity_type) from `logAudit('create', 'lesson', ...)` rows (which always had singular entity_type). Both would be at risk of being rewritten to `'insert'`.

**Solution:** run action backfill first, filtering on the *plural* entity_type values. Only triggers wrote those plurals, so the filter cleanly isolates trigger rows. Then run entity_type backfill.

```sql
-- Step 1: action backfill (filter on plural entity_type — only triggers wrote those)
UPDATE audit_log SET action = 'insert'
  WHERE action = 'create'
    AND entity_type IN (
      'students','lessons','invoices','payments','org_memberships',
      'ai_action_proposals','teachers','internal_messages'
    );

-- Step 2: entity_type backfill (plural → singular)
UPDATE audit_log SET entity_type = 'student'             WHERE entity_type = 'students';
-- ...etc
```

```typescript
export function getChangeDescription(entry: AuditLogEntry): string {
  const entityLabel = getEntityLabel(entry.entity_type);
  const actionLabel = getActionLabel(entry.action).toLowerCase();

  // INSERT path — trigger-emitted CRUD insert. 'create' is also accepted as
  // a logAudit() domain-verb fallback, since some callers in src/ deliberately
  // write 'create' to express "user created this" rather than CRUD insert.
  if ((entry.action === 'insert' || entry.action === 'create') && entry.after) {
    if (entry.entity_type === 'student') {
      const name = `${entry.after.first_name || ''} ${entry.after.last_name || ''}`.trim();
      return `${entityLabel} "${name}" was created`;
    }
    if (entry.entity_type === 'lesson') {
      return `Lesson "${entry.after.title || 'Untitled'}" was scheduled`;
    }
    if (entry.entity_type === 'invoice') {
      return `Invoice ${entry.after.invoice_number || ''} was created`;
    }
    if (entry.entity_type === 'payment') {
      const amount = ((entry.after.amount_minor as number) || 0) / 100;
      return `Payment of £${amount.toFixed(2)} was recorded`;
    }
    if (entry.entity_type === 'org_membership') {
      return `Member was added with role "${entry.after.role}"`;
    }
  }

  if (entry.action === 'update') {
    if (entry.entity_type === 'invoice' && entry.before && entry.after) {
      if (entry.before.status !== entry.after.status) {
        return `Invoice ${entry.after.invoice_number || ''} status changed from "${entry.before.status}" to "${entry.after.status}"`;
      }
    }
    if (entry.entity_type === 'lesson' && entry.before && entry.after) {
      if (entry.before.status !== entry.after.status) {
        return `Lesson "${entry.after.title || 'Untitled'}" was ${entry.after.status}`;
      }
    }
    if (entry.entity_type === 'org_membership' && entry.before && entry.after) {
      if (entry.before.role !== entry.after.role) {
        return `Member role changed from "${entry.before.role}" to "${entry.after.role}"`;
      }
    }
  }

  if (entry.action === 'delete') {
    if (entry.entity_type === 'student' && entry.before) {
      const name = `${entry.before.first_name || ''} ${entry.before.last_name || ''}`.trim();
      return `${entityLabel} "${name}" was deleted`;
    }
  }

  return `${entityLabel} was ${actionLabel}`;
}
```

```markdown
## Audit triggers

### When to add one

Add an audit trigger to any table where:
- Out-of-band edits (service-role mutations, RLS-bypass, manual SQL) need a trail.
- Manual `logAudit()` calls in `src/` cover only a fraction of write paths.
- The data is money-, identity-, or policy-adjacent.

### Which pattern

Two helpers exist; which to use depends on whether the table has `org_id`:

- **`log_audit_event_singular(<entity_type>)`** — for org-scoped tables with
  a NOT NULL `org_id`. Writes to `audit_log`. Used by 16 T01-P1 triggers and
  the 8 T01-P3-rewritten triggers. Pass entity_type as a quoted string in
  TG_ARGV[0]. Singular form (`'student'`, not `'students'`).

- **`log_profile_change()`** (custom) — for per-user tables without `org_id`.
  Writes to `platform_audit_log`. Used by `audit_profile_changes` (T01-P2).
  If a new per-user audit need arises, write a new custom function following
  this pattern, not a generalisation of `log_audit_event_singular`.

`audit_attendance_changes()` is a third custom helper preserved for historical
reasons. New tables should not use it; use `log_audit_event_singular`.

### entity_type convention

- Always **singular** (`'invoice'`, not `'invoices'`).
- Always **snake_case** (`'org_membership'`, not `'orgMembership'`).
- Match the table's logical entity name, not the table name. `teachers` table
  → entity_type `'teacher'`. `org_memberships` table → entity_type
  `'org_membership'`.

### action verb convention

- **CRUD triggers** write `LOWER(TG_OP)`: `'insert'` | `'update'` | `'delete'`.
- **Domain-event writes** from `logAudit()` callers in `src/` and SECURITY
  DEFINER RPCs in migrations write rich domain verbs (e.g. `'cancel'`,
  `'streak_milestone'`, `'student.created'`).
- The two action-verb spaces coexist intentionally. The frontend display
  layer (`src/hooks/useAuditLog.ts`) maps both `'insert'` and `'create'` to
  the "Created" label so domain-verb writes render consistently.

### Idempotency

Every audit-trigger migration must be safe to re-apply:
- `CREATE OR REPLACE FUNCTION` for helpers.
- `DROP TRIGGER IF EXISTS` ahead of `CREATE TRIGGER`.
- Backfill UPDATEs are naturally idempotent given WHERE-clause filtering.
```
